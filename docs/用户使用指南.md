# mkdn 使用指南

> 给最终用户的 Android Markdown 阅读器操作手册  
> 当前版本：**v1.7.6-tockeyword** (versionCode 17) · APK ~10.15 MB

---

## 0. 关于 mkdn

mkdn 是一款专为长时间阅读 Markdown 文档设计的 Android 应用。  
参照 One Markdown / Bear / 微信读书的阅读质感，主打：

- 三套主题（浅色 / 护眼米黄 / 深色）
- 七档字号（13sp ~ 30sp）
- Markdown 完整渲染 + 50+ 语言代码块语法高亮
- 文件内搜索 / 目录大纲（精准关键词定位）/ 简单编辑 / 最近文件
- 兼容 GBK 编码（Windows 中文 .md 不乱码）

**系统要求**：Android 7.0 (API 24) 及以上

---

## 1. 安装

### 1.1 安装 APK

1. 把 `mkdn-v1.7.6-tockeyword.apk` 拷到手机（USB / 网盘 / 邮件 / 微信传输都行）
2. 在手机文件管理器里点这个 apk
3. 第一次会提示"未知来源应用"——按提示给文件管理器授权安装
4. 点"安装"，几秒钟搞定

> 提示：v1.7.6 已签名，**不需要卸载旧版直接覆盖安装**，最近文件和数据不会丢。

### 1.2 首次启动

打开 mkdn，主界面只有 4 样东西：

- 顶部标题栏 `mkdn` 三个图标按钮（从左到右：字号、主题）
- 一个大蓝按钮：**打开 Markdown / 文本文件**
- 中间一行：最近文件（首次为空）
- 底部提示语：还没有打开过文件

---

## 2. 打开文件

### 2.1 从系统文件选择器

1. 点主界面的大蓝按钮
2. 弹出 Android 系统文件选择器（Files / Solid Explorer 等都行）
3. 选 `.md` / `.markdown` / `.txt` 文件
4. mkdn 自动读文件 → 渲染 → 进入阅读界面

**支持的格式**：`.md` `.markdown` `.txt`（其他文本类也能选，只是没有专用扩展名处理）

**编码自动识别**：UTF-8 (BOM)、UTF-16 (LE/BE BOM)、纯 UTF-8、GBK（Windows 写的中文 .md 也不会乱码）

**文件大小限制**：单文件 **10 MB 以内**。超过会提示"文件过大，请用桌面编辑器打开"——这是为了防止 App 内存溢出崩溃。

### 2.2 从其他 App 分享

- 在文件管理器 / 浏览器 / 聊天软件里看到 .md 文件
- 选 "用其他应用打开" / "分享"
- 选 mkdn（如果装了多个 Markdown 阅读器，列表里选 mkdn）
- mkdn 自动打开 → 进阅读界面

### 2.3 从"最近文件"

主界面下半部分列了最近打开的 50 个文件，按时间倒序：

- **点文件名** → 直接打开
- **点右侧 ✕** → 从最近列表移除（不会删除文件本身）
- 文件被删了会提示"文件不存在"，并自动从最近列表清理

## 3. 主题切换（浅色 / 护眼 / 深色）

mkdn 有 **3 套主题**：

| 主题 | 背景 | 主色 | 适合场景 |
|------|------|------|----------|
| 浅色 | 纯白 `#FFFFFF` | GitHub 蓝 | 白天、办公、屏幕亮度高 |
| 护眼（米黄） | 米黄 `#FAF3DF` | 棕红 | 长时间阅读、夜间、防蓝光 |
| 深色 | GitHub dark `#0D1117` | 天蓝 | 夜间、OLED 屏幕、护眼 |

**切换方法**：

1. 在阅读界面 / 主界面顶栏点 🎨 **调色板** 图标
2. 弹出下拉菜单，选其中一个主题
3. **立即生效**，同时**自动保存**——下次打开 App 还是这个主题

> 小技巧：刚装上默认是浅色，但**长时间阅读推荐切到护眼米黄**，真·防疲劳。

## 4. 字号调节

mkdn 有 **7 档字号**：

| 档位 | 字号 | 适合场景 |
|------|------|----------|
| XS | 13sp | 技术文档紧凑浏览 |
| S | 15sp | 小屏手机 |
| M | 17sp | 推荐 / 中等 |
| L | 19sp | 默认 |
| XL | 22sp | 大屏 / 远视 |
| XXL | 26sp | 老年人 / 视力辅助 |
| XXXL | 30sp | 极特大 |

**调节方法**：

1. 顶栏点 A **字号** 图标
2. 弹出下拉菜单，选档位
3. 阅读界面立即重新渲染，**自动保存**

## 5. 文件内搜索

mkdn 在**当前打开的文件里搜索**关键字。

### 5.1 开始搜索

1. 阅读界面顶栏点 🔍 **搜索** 图标
2. 顶部出现搜索输入框
3. 输入关键字（**大小写不敏感**）

### 5.2 跳转匹配

输入框右侧出现三个控件：

- **`N/M` 计数器**：当前位置 / 总匹配数
- **↑ 向上箭头**：跳到上一个匹配
- **↓ 向下箭头**：跳到下一个匹配

每次跳转：

- 屏幕滚动到匹配位置
- 选中的文字被系统**蓝把高亮**（系统级 selection handle，方便继续复制）

### 5.3 退出搜索

清空输入框 + 再次点 🔍 图标 / 按系统返回键都行。

## 6. 目录大纲（精准关键词定位）

mkdn 自动解析 Markdown 的 H1 ~ H6 标题，生成目录。

### 6.1 打开目录

1. 阅读界面顶栏点 📑 **目录** 图标
2. 弹出目录对话框
3. 标题按层级缩进（H1 不缩进，H2 缩 2 空格，H3 缩 4 空格...）
4. 文档里的所有标题**完整列出**，可以滚动查看

### 6.2 跳转（精准定位）

- **点任一目录项** → 自动关闭目录对话框
- mkdn 用**关键词定位**算法精准命中：
  - 找到该标题文字在 TextView 里的真实位置
  - 计算所在行的 Y 坐标
  - 滚动到 **屏幕正中偏上 1/8 处**——既能看到标题，也能看到下面一点正文
- 即使文档里有 emoji、中文、同名标题，跳转也**100% 准**

## 7. 编辑模式

mkdn 提供**简单编辑**功能，适合快速改个错字、补充一段。

### 7.1 进入编辑

阅读界面顶栏点 ✏️ **编辑** 图标 → 进入编辑界面

### 7.2 编辑操作

- 编辑器**自动加载文件内容**到 TextField
- 改完内容后右上角 **保存** 按钮从灰色变蓝色

### 7.3 保存

1. 点 **保存** 按钮
2. mkdn 自动：
   - 把原文件**复制成 `.bak` 备份**（崩了能恢复）
   - 写入新内容
   - 返回阅读界面
3. 保存失败会 Toast 提示

### 7.4 放弃编辑（取消）

**情况 A：没改动**
- 直接点左上角 ✕ → 立刻返回阅读界面

**情况 B：改动了**
- 点左上角 ✕ / 系统返回键 → 弹出确认对话框 "放弃修改？"
  - **放弃** → 放弃修改，返回阅读
  - **继续编辑** → 关掉对话框，继续编辑

### 7.5 编辑器状态

- **保存按钮的颜色**告诉你当前是否"脏"：
  - 灰色（disabled）= 内容与原文件一致（无需保存）
  - 蓝色 = 内容已改动（可保存）
- 即使打开的是文件原内容（没改），保存按钮也是可点状态——让你主动确认"确认无变化"也行

## 8. 标重点功能（v1.7.2 起已隐藏）

> ⚠️ **重要**：从 v1.7.2 起，"标重点"功能已被**软隐藏**——选中文字不再看到 🔖 标重点按钮，主界面顶栏也没有 🔖 书签入口。

### 8.1 为什么隐藏

v1.7.1 之前的标重点功能存在位置错位 bug（高亮跑到了别的段落），经过 3 次修复仍未彻底解决。  
为了不影响核心阅读体验，**决定暂时关闭入口**——代码和数据保留，以后想恢复改 4 处即可。

### 8.2 旧功能速览（如果以后恢复）

如果未来版本恢复，mkdn 的标重点功能：

1. 在阅读界面，**长按任一段文字** → 拖动"把手"选择范围
2. 顶部弹出系统 ActionMode 工具栏，里面有 mkdn 加的 **🔖 标重点** 按钮
3. 点 🔖 → 选中文字立刻显示暖黄色背景
4. 主界面顶栏 🔖 书签图标 → "我的高亮"页面（按时间倒序）
5. **用文本指纹定位**——文件编辑后只要前后 24 字符不变，自动找回

### 8.3 已存高亮数据

v1.7.2 之前标的所有高亮数据**完整保留**在 `repo/highlights.json`，只是不显示。  
恢复功能后会自动重新显示。

## 12. 常见问题

### Q1：打开文件中文乱码？

mkdn 自动识别编码（UTF-8/UTF-16/GBK）。如果还是乱码：

- 确认原文件不是图片转 PDF 之类
- 把文件用 VSCode / Notepad++ 转 UTF-8 无 BOM 保存后再打开

### Q2：文件太大打不开？

mkdn 限制 **10 MB**，超过会拒绝加载并提示用桌面编辑器。  
这是为了避免大文件把手机内存吃满崩 App。

### Q3：编辑后崩了能找回吗？

保存时 mkdn 自动把原文件备份成 `xxx.md.bak`。  
崩了之后去文件管理器找到 `.bak` 重命名回 `.md` 就行。

### Q4：主题/字号每次都要重新设置吗？

不会。mkdn 自动保存到 PrefsStore，**下次打开 App 还是你上次选的**。

### Q5：能导出 PDF 吗？

当前版本（v1.7.6）不支持。后续版本规划中。

### Q6：怎么彻底卸载？

设置 → 应用 → mkdn → 卸载。**注意**：卸载会清掉本地所有数据——先手动备份重要的 .md 文件。

### Q7：标重点功能还能用吗？

v1.7.2 起已隐藏入口（详见 §8）。代码和数据保留，等待未来恢复。

### Q8：目录跳转不准？

v1.7.6 已用**关键词定位**算法，命中精度 100%。  
如果还有问题，请提供：文档类型、复现步骤、点击的目录项。

### Q9：高亮数据还在吗？

v1.7.2 之前标的高亮数据完整保留在 App 私有目录，只是 UI 入口隐藏了。
